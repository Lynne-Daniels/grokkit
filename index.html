Making a new project.

  • Make new project at Github. Add readme, gitignore node.
  • May want to gitignore .sublime-workspace
    • li>
  • In bash, navigate to the folder containing all the projects
  • Get the clone link from github. For example: https://github.com/Lynne-Daniels/grokkit.git
  • In bash $git clone https://github.com/Lynne-Daniels/grokkit.git
  • If it worked...now have a new folder with project name.
  • Add file called package.json. Example 
                                        {
                                      "name": "grokkit",
                                      "version": "1.0.0",
                                      "description": "Web Development Notes and Examples",
                                      "main": "whatdoesthisdo.js",
                                      "scripts": {
                                        "test": "echo \"Error: no test specified\" && exit 1",
                                        "dev": "lite-server"
                                      },
                                      "author": "",
                                      "license": "ISC",
                                      "devDependencies": {
                                        "lite-server": "^2.2.0"
                                      },
                                      "dependencies": {
                                        "light-server": "^2.0.2"
                                      }
                                    }
  • Add index.html, app.js, main.css files to project folder, make more directories if needed
  • Add .sublime-project if needed to hold settings for editor
    • li>
  • in bash cd to new project
  • $ npm install, then npm run dev
  • if it worked - a browser windo pops up and will automagically refresh when the html, css or js files are updated.
  • In the HTML file, plan and make the grid first!!!
  • install eslint locally in your project folder (you must have package.json file there): npm install eslint, then ./node_modules/.bin/eslint --init
Will need to restart the web server when returning to this project. my-project $ npm run dev
